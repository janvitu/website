## Layout Contract

### Terminal Card Structure

```
[data-theme="terminal"] body
  └── base-grid (12-column)
       └── terminal card  (col-start-3 col-end-11)
            ├── title-bar         (fixed height: 40px)
            │    ├── traffic-light dots (red/yellow/green, 12px each, 8px gap)
            │    └── path label   (~/how-i-ai, monospace 14px, centered)
            ├── content-area      (flex-1, overflow-y-auto)
            │    └── {{ .Content }} / tab panels (Phase 4)
            └── status-bar        (fixed height: 36px, pinned bottom)
                 ├── left:   ● claude-code
                 ├── center: 1/4  (accent-colored)
                 └── right:  Opus 4.6 · Thinking...
```

### Height Calculation

Terminal card height: `calc(100dvh - var(--header-height) - var(--footer-height) - 96px)`

- `96px` = vertical breathing room (48px top + 48px bottom margin between card and header/footer)
- Minimum height: `400px` (prevents card from collapsing on very short viewports)
- `dvh` (dynamic viewport height) used over `vh` to avoid mobile browser chrome issues

If CSS variable header/footer heights are not already declared in Phase 1 output, the executor must measure and hardcode the values as a Tailwind arbitrary value (e.g., `calc(100dvh - 64px - 48px - 96px)`) after inspecting the rendered layout.

### Card Visual Properties

- Border: `1px solid var(--color-border)` on all four sides
- Border-radius: `8px` (subtle rounding — terminal window feel, not pill shape)
- Background: `var(--color-bg-surface)` (`#252525`)
- Box-shadow: none on card itself (glow is handled by the pseudo-element behind it)

### Background Glow (TERM-04)

Implementation: `::before` pseudo-element on the terminal card, or a sibling `<div>` placed behind the card via `z-index: -1`.

Properties:
- Shape: `radial-gradient(ellipse 80% 60% at 50% 50%, rgba(255, 67, 9, 0.06) 0%, transparent 70%)`
- Size: 120% of card width, 80% of card height — extends slightly beyond card edges
- Position: centered behind the card
- Pointer events: none
- z-index: -1 (behind card, in front of page background)

This reads as a monitor backlight — warm tint, not a ring or border glow. Keep opacity at or below 0.08; higher values make the accent color intrusive on the dark background.

Source: CONTEXT.md D-12 (glow at Claude's discretion — design decision made here)
