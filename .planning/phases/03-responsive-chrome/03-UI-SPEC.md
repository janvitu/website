## Responsive Layout Contract

### Grid (source: CONTEXT.md D-01)

| Breakpoint | Column span | Tailwind classes |
|------------|-------------|------------------|
| Mobile (default) | col 2–12 (1 gutter column each side) | `col-start-2 col-end-12` |
| Desktop (md: 768px+) | col 3–11 (2 gutter columns each side) | `md:col-start-3 md:col-end-11` |

### Title Bar (source: CONTEXT.md D-04)

| Breakpoint | Layout | Notes |
|------------|--------|-------|
| Mobile (default) | `justify-between`: dots left, `~/how-i-ai` right-aligned | Drop the 56px invisible spacer div |
| Desktop (md: 768px+) | Centered: `flex-1 text-center` path + 56px spacer | Existing layout, unchanged |

Tailwind implementation: apply `flex justify-between md:justify-start` on title bar container; on path span apply `md:flex-1 md:text-center`; on spacer div apply `hidden md:block`.

Height 40px and padding `0 16px` are unchanged at all breakpoints.

### Status Bar (source: CONTEXT.md D-02, D-03)

| Element | Mobile | Desktop |
|---------|--------|---------|
| Green indicator dot + `claude-code` label | Visible | Visible |
| Pagination `1/4` (accent orange) | Visible | Visible |
| Model text ("Opus 4.6 · Thinking...") | Hidden | Visible |

Tailwind implementation: add `hidden md:inline` to the model text `<span>`.

Height 36px and padding `0 16px` are unchanged at all breakpoints.

### Terminal Height (source: CONTEXT.md D-06, D-07, D-08)

| Breakpoint | Value | Rationale |
|------------|-------|-----------|
| Mobile (default) | `80svh` | `svh` avoids layout jank from mobile browser address bar show/hide; `~80svh` keeps header and footer visible |
| Desktop (md: 768px+) | `calc(100dvh - 80px - 128px - 96px)` | Existing `--terminal-height` value, unchanged |
| Fallback (all) | `min-height: 65vh` | Safety net for browsers without svh support |

Implementation: override `--terminal-height` inside a `@media (max-width: 767px)` block in `tailwind.css` under `[data-theme="terminal"]`:
```css
@media (max-width: 767px) {
  [data-theme="terminal"] {
    --terminal-height: 80svh;
  }
}
```

### Wrapper Padding (source: CONTEXT.md D-09, Claude's discretion)

| Breakpoint | padding-top | padding-bottom |
|------------|-------------|----------------|
| Mobile (default) | 48px | 24px |
| Desktop (md: 768px+) | 96px | 48px |

Implementation: convert the inline `style="padding-top: 96px; padding-bottom: 48px"` on the `.base-grid` wrapper to Tailwind utility classes: `pt-12 pb-6 md:pt-24 md:pb-12` (using Tailwind 4 default 4px base: 12×4=48px, 6×4=24px, 24×4=96px, 12×4=48px).

---

## Breakpoint Decision (source: CONTEXT.md Claude's Discretion)

Use `md:` (768px) as the single breakpoint for all responsive overrides in this phase. Rationale: 375px (iPhone SE) is well below 768px, and the `md:` breakpoint cleanly separates handheld (mobile + large phone) from tablet/desktop. The `sm:` breakpoint at 640px would create a narrow band of partially-responsive state.

---

## Implementation Strategy (source: CONTEXT.md Claude's Discretion)

Use Tailwind responsive prefix classes in `terminal-frame.html` HTML for all structural changes (grid columns, title bar layout, status bar visibility, wrapper padding). Use a `@media` block in `tailwind.css` only for the `--terminal-height` CSS custom property override, following the established `@media (prefers-reduced-motion)` pattern in that file.
