## Spacing Scale

Declared values (multiples of 4, consistent with existing terminal chrome):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot separators |
| sm | 8px | Gap between traffic-light dots, status bar indicator gap |
| md | 16px | Horizontal padding: tab strip, title bar, status bar |
| lg | 24px | Horizontal padding: content area |
| xl | 32px | Vertical padding: content area top/bottom |
| 2xl | 48px | H1 intro block vertical padding (mobile: `pt-16`, desktop: `pt-24`) |
| 3xl | 64px | not used in this phase |

Exceptions:
- Tab button vertical padding: 10px top/bottom (not on scale — matches existing title bar 40px height rhythm; keep at 10px)
- Tab strip height: 40px — matches title bar row height for visual consistency
- Status bar height: 36px — pre-existing, do not change
- Title bar height: 40px — pre-existing, do not change

Source: terminal-frame.html (existing padding values), CONTEXT.md D-02, RESEARCH.md Pattern 2

- ARIA: `role="tablist"` on container, `aria-label="How I AI sections"`

### Tab Button (×4)

Tabs: **My Stack**, **Setup**, **Workflow**, **Tips** (in this order, left to right)

| State | Text color | Bottom border | Background |
|-------|-----------|---------------|------------|

### Tab Panel

- All 4 panels in DOM at page load (no lazy loading). Source: CONTEXT.md D-06.
- Active panel: no `hidden` attribute. Inactive panels: `hidden` HTML attribute (JS toggles it).
- ARIA per panel: `role="tabpanel"`, `id="panel-{id}"`, `aria-labelledby="tab-{id}"`, `data-tab-id="{id}"`
- IDs: `panel-my-stack`, `panel-setup`, `panel-workflow`, `panel-tips`
- Rendered via Hugo shortcode `{{< tab id="..." label="..." >}}...{{< /tab >}}`
- First tab (`my-stack`) must NOT have `hidden` on page load — shortcode uses `active="true"` param or TS removes it on init
- Source: CONTEXT.md D-05, D-06, RESEARCH.md Pattern 1, Pitfall 3

## Interaction Contract

### Tab Click

1. User clicks a tab button
2. All tab buttons: set `aria-selected="false"`, text color to `var(--color-text-muted)`, bottom border color to `transparent`
3. Clicked tab button: set `aria-selected="true"`, text color to `var(--color-text)`, bottom border color to `var(--color-accent)`
4. All tab panels: set `hidden` attribute
5. Target panel (matching `data-tab` value): remove `hidden` attribute
6. No URL hash update (Phase 5). No transition animation (Phase 5).

Source: CONTEXT.md D-07, RESEARCH.md Pattern 3

### Page Load (initial state)

- My Stack tab is active: `aria-selected="true"`, full-color label, accent bottom border
- My Stack panel is visible: no `hidden` attribute
- All other tabs/panels: inactive/hidden
- Status bar `1/4` remains hardcoded static — no dynamic update

Source: CONTEXT.md D-08, RESEARCH.md Pitfall 3

### Keyboard / Accessibility

- Tab buttons are native `<button>` elements — keyboard focusable by default
- Focus ring: `outline: 2px solid #ffffff` at `outline-offset: 2px` (pre-existing `[data-theme="terminal"] :focus-visible` rule in tailwind.css — no new CSS needed)
- Arrow key tab switching: NOT in scope for Phase 4 (deferred to v2 TABS-04)

## Implementation Notes for Executor

1. **Remove existing H1 from `_index.en.html`** before replacing with shortcodes — current line 6 has `<h1 class="text-4xl...">How I AI</h1>`. Leaving it causes two H1 elements on the page. Source: RESEARCH.md Pitfall 5.

2. **Use `{{ .Inner | safeHTML }}`** in the shortcode template, not `{{ .Inner }}` — prevents HTML entity escaping of inner HTML tags. Source: RESEARCH.md Pitfall 2.

3. **Tab strip `shrink-0` is mandatory** — the terminal frame is a flex column with fixed height. Without `shrink-0` the tab strip will compress and the status bar may be pushed off-screen. Source: RESEARCH.md Pitfall 4.

4. **Active state via inline styles** (not Tailwind utility classes) — consistent with existing terminal-frame.html pattern where all chrome elements use inline `style=""`. The TS module updates `button.style.borderBottomColor` and `button.style.color` directly. Source: RESEARCH.md Open Question 1.

5. **New TS module `hugo/assets/ts/tabs.ts`** — do not put tab switching logic inline in `index.ts`. Import via `import { initTabs } from "./tabs"` and call inside the existing `DOMContentLoaded` listener. Source: RESEARCH.md Pattern 3, Architecture Patterns.
