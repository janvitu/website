## Typography

Phase 1 establishes the type foundation for the dark scoped context. The existing site globals (Gilroy on `html`) carry over; this phase adds monospace for terminal identity and declares the dark-scoped text colors.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Gilroy | 16px | 400 (Regular) | 1.5 | General page text on dark background |
| Label | Gilroy | 14px | 600 (SemiBold) | 1.4 | UI labels, status bar text |
| Heading | Gilroy | 20px | 600 (SemiBold) | 1.2 | Section headings within page |
| Terminal | system monospace | 13px | 400 (Regular) | 1.4 | Terminal chrome text (title bar, status bar), code spans |

Notes:
- Only 2 weights in use: 400 (Regular) and 600 (SemiBold). Source: codebase/CONVENTIONS.md pattern.

- The existing h1/h2 clamp rules from `tailwind.css` are NOT used on this page — those are for the light site. This page uses fixed sizes within the terminal frame.

## Color

### Dark Theme Palette

Applied exclusively within `[data-theme="terminal"]`. Light theme remains untouched.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a1a` | Page background (terminal outer fill) |
| Secondary (30%) | `#252525` | Terminal frame surface, content area background |
| Border / separator | `#3a3a3a` | Terminal window border, divider lines, grid line override |
| Text primary | `#e0e0e0` | Body text, title bar text, tab labels |
| Text muted | `#888888` | Status bar secondary text, placeholder labels |
| Accent (10%) | `#ff4309` | See reserved list below |
| Focus ring | `#ffffff` | Focus outline on dark background (white is safer than accent on dark) |

Source: Research proposed values in research/STACK.md — `#1a1a1a`, `#252525`, `#3a3a3a`, `#e0e0e0`, `#888`. Accent `#ff4309` from existing `tailwind.css` `--color-accent`.

### Accent Reserved For (Phase 1 scope only)

Accent (`#ff4309`) is reserved **only** for:

1. Focus ring outline on interactive elements — when white outline is insufficient contrast
2. Hover state indicator (future phases may add more; Phase 1 establishes the token only)

Accent is NOT used for: general text, backgrounds, borders, grid lines, or decorative elements.

### Grid Line Override (THEME-02)

Existing grid partial uses `border-neutral-300` (approximately `#d4d4d4`). Within `[data-theme="terminal"]`, override grid line color to `#3a3a3a` (matches `--color-border` token). Grid lines must remain visible but subtle — not dominant.

CSS custom property map for dark scope:

```css
[data-theme="terminal"] {
  --color-bg: #1a1a1a;
  --color-bg-surface: #252525;
  --color-border: #3a3a3a;
  --color-text: #e0e0e0;
  --color-text-muted: #888888;
  --color-accent: #ff4309;
}
```

## Focus Indicators (THEME-03)

Phase 1 requirement: focus indicators must be visible on dark background.

| Element | Focus Style |
|---------|-------------|
| Interactive elements (links, buttons) | `outline: 2px solid #ffffff; outline-offset: 2px` |
| Accent-colored interactive elements | `outline: 2px solid #ff4309; outline-offset: 2px` |

Implementation: Use `focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-white` Tailwind classes. Match pattern used in existing `link-strikethrough` which uses `focus-visible:outline-accent`. On dark, prefer white as the primary focus color (higher contrast against `#1a1a1a`).
