  <action>
Append the following CSS block to hugo/assets/css/tailwind.css, placed just ABOVE the closing `@media (prefers-reduced-motion)` block so it reads logically: components, then dark theme, then reduced motion.

Add this exact CSS:

```css
/* DARK THEME -- scoped to /how-i-ai terminal page only */
[data-theme="terminal"] {
	--color-bg: #1a1a1a;
	--color-bg-surface: #252525;
	--color-border: #3a3a3a;
	--color-text: #e0e0e0;
	--color-text-muted: #888888;

	background-color: #1a1a1a;
	color: #e0e0e0;
}

[data-theme="terminal"] .grid > div {
	border-color: #3a3a3a;
}

[data-theme="terminal"] :focus-visible {
	outline: 2px solid #ffffff;
	outline-offset: 2px;
}
```

Key details:
- Use tabs for indentation (project convention: useTabs: true)
- The [data-theme="terminal"] selector has specificity (0,1,0) which beats the body element selector (0,0,1) that sets @apply bg-neutral-100 - so the dark background-color overrides the light one correctly
- The grid override targets .grid > div because grid.html renders a div with class "grid grid-cols-12" containing child divs with border-neutral-300. The .grid > div selector overrides the Tailwind border-neutral-300 class within the dark scope
- The :focus-visible override provides a blanket white focus ring for all interactive elements on the dark page, replacing the accent-colored ring from the light theme
- Do NOT modify any existing CSS rules. Do NOT change the @theme block, html, body, or component rules
